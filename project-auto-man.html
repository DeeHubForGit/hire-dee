Auto-Man: Driving School Website

A full-featured web application built to support a local driving school with online lesson bookings, client management and calendar integration.

  • Online Booking: Students can book lessons directly through the website with real-time availability.
  • Client Portal: Learners can view upcoming lessons, complete intake, update details and messages.
  • Admin Dashboard: Admin interface to manage bookings, clients and schedules in one place.
  • Automated Reminders: SMS and email confirmations help reduce missed lessons.
  • Online Payments: Stripe integration allows clients to pay securely online for selected bookings and driving test appointments.
  • Calendar Sync: Bookings automatically appear in Google Calendar to avoid double bookings.
  • Mobile Friendly: Designed for real users on phones, tablets and desktop devices.

Technical Information

  • Architecture: Full-stack web application with separated public site, client portal and admin dashboard layers.
  • Backend: Supabase (PostgreSQL) with row-level security, structured schema design and controlled data access.
  • Authentication: Secure user authentication with protected routes and role-based access control.
  • Calendar Integration: Bi-directional Google Calendar sync to manage bookings and prevent scheduling conflicts.
  • Address Validation: Google Maps API integration to validate and standardise pickup addresses, reducing incorrect bookings and preventing wasted instructor travel time.
  • State Management: Persistent client and booking state handling with validation to maintain data integrity.
  • Messaging Integrations: Automated email (Resend) and SMS (ClickSend) notifications triggered by booking and payment events.
  • Payment Integration: Stripe checkout integration supports secure online payments for selected services, with payment status flowing back into the booking system.
  • Error Handling: Defensive validation and structured error responses to prevent broken booking flows.
  • Codebase: 81 source files, approximately 30k lines across 43 HTML, 30 TypeScript, 9 JavaScript, 1 CSS and 1 SQL file.

Main Website

  • Service pages describing each offering in detail including pricing.
  • Supporting pages for FAQs, Service Areas and Contact Us.
  • Clear calls-to-action to guide visitors into booking.
  • Built to support real users, on mobile and desktop.

Key feature: Booking flow

Users can select a service, choose an available time and complete a booking, with calendar sync ensuring availability stays accurate.

Client Portal

  • View, correct or cancel upcoming bookings.
  • Intake form for new users to record important information like their permit number.
  • Update personal details without needing admin help.
  • View any messages sent through the Contact Us page.
  • Designed to reduce back-and-forth and support self-service.

Key feature: My Bookings

A simple view of bookings with the information clients actually need, without exposing admin-only complexity.

Admin Dashboard

  • Calendar: Calendar-first workflow for managing lessons and availability. Add, edit, reschedule and cancel bookings and personal appointments.
  • Bookings: View all bookings with validation, status flags and operational checks (e.g. missing info, payment required).
  • Cancellations: Track cancelled lessons and refund status with clear “action required” visibility.
  • Clients: Manage client records and key details used across bookings and communications. Complete intake for clients who are unable to do it themselves.
  • Client Progress: Tick-off skill progress during lessons with auto-save for fast, consistent tracking.
  • Reporting: Simple reports for operational visibility without relying on spreadsheets.
  • Send SMS & Email: Built-in tools to contact clients quickly from the dashboard.
  • Contact Messages: View and respond to “Contact Us” enquiries, with unread tracking.

Key feature: Admin calendar

A central calendar view to manage bookings and keep scheduling aligned with Google Calendar.
